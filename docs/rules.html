嚴重度說明
ERROR
結構錯誤 資源不符合 FHIR R4 規範，SMART App 或 FHIR 伺服器可能無法正常讀取。應修正。
WARNING
不完整 資源可以被解析，但缺少建議欄位或不符合最佳實務。建議修正。
INFO
提示 台灣 NHI 特有建議，或代碼未收錄在已知集合中。可依需求決定是否處理。

離線優先設計 所有驗證邏輯均在瀏覽器本地執行，不呼叫任何外部 API。LOINC 代碼集、HL7 value sets 全部內嵌於程式中，資料不會離開你的裝置。
Patient（病人）
基本病人身份資訊，是所有其他資源的參照根基。
Rule ID檢查項目說明嚴重度
pat-identifier-requiredidentifier 非空必須至少一個識別碼（身分證號、病歷號），FHIR 接收方才能辨識病人ERROR
pat-name-requiredname 非空必須包含至少一個姓名ERROR
pat-gender-invalidgender 值域只允許 male、female、other、unknownERROR
pat-name-use-invalidname[*].use 值域必須在 HL7 name-use value set 內（usual/official/temp/nickname/anonymous/old/maiden）WARNING
pat-birthdate-formatbirthDate 格式必須符合 YYYY-MM-DD，不能使用民國年WARNING
Observation（觀察值 / 檢驗數值）
個別的檢驗數值、生命徵象等，是資料量最多的資源類型。
Rule ID檢查項目說明嚴重度
obs-status-requiredstatus 必填且值域正確表示觀察值狀態（final/preliminary/amended 等），必填ERROR
obs-code-requiredcode.coding 非空至少一個 coding 標識此觀察項目（通常為 LOINC）ERROR
obs-subject-requiredsubject.reference 格式必須格式為 Patient/<id>ERROR
obs-comparator-invalidvalueQuantity.comparator 值域只允許 <、<=、>=、>ERROR
obs-refrange-numericreferenceRange.low/high.value 為數字參考區間的上下界必須是數值ERROR
obs-category-missingcategory 存在建議填寫分類（如 laboratory、vital-signs），方便篩選WARNING
obs-value-missing至少一個 value[x] 或 dataAbsentReason若無數值應填寫缺值原因WARNING
obs-ucum-systemvalueQuantity.system 為 UCUM應填 http://unitsofmeasure.orgWARNING
obs-ucum-code-emptyvalueQuantity.code 非空指定 UCUM system 時，單位代碼不能為空WARNING
obs-loinc-unknownLOINC code 已知LOINC 代碼不在驗證器收錄集，建議至 loinc.org 確認INFO
DiagnosticReport（檢驗報告）
彙整多個 Observation 的檢驗報告（如一次抽血的完整結果）。
Rule ID檢查項目說明嚴重度
dr-status-requiredstatus 必填且值域正確必填，表示報告狀態ERROR
dr-code-requiredcode 存在標識報告類型，必填ERROR
dr-subject-requiredsubject.reference 格式必須格式為 Patient/<id>ERROR
dr-issued-missingissued 存在（status=final）已完成的報告應記錄出具時間；EHR-FHIR-BRIDGE 舊版 mapper 已知問題，新版已修正WARNING
dr-result-ref-invalidresult[*].reference 格式必須格式為 Observation/<id>WARNING
MedicationRequest（藥囑）
開立的藥物處方，包含藥品代碼、用法用量。
Rule ID檢查項目說明嚴重度
med-status-requiredstatus 必填且值域正確必填（active/completed/cancelled 等）ERROR
med-intent-requiredintent 必填且值域正確必填（order/plan 等），表示藥囑性質ERROR
med-medication-requiredmedicationCodeableConcept 有 coding必填，標識藥品ERROR
med-subject-requiredsubject.reference 格式必須格式為 Patient/<id>ERROR
med-dosage-text-missingdosageInstruction[*].text 存在建議填寫文字用藥說明，方便人工閱讀WARNING
Condition（診斷）
病人的疾病診斷記錄（通常含 ICD-10 代碼）。
Rule ID檢查項目說明嚴重度
con-status-requiredclinicalStatus 存在且有效必填，表示疾病目前狀態（active/resolved 等）ERROR
con-code-requiredcode 存在必填，標識診斷（ICD-10）ERROR
con-subject-requiredsubject.reference 格式必須格式為 Patient/<id>ERROR
AllergyIntolerance（過敏）
病人的過敏或不耐受記錄。
Rule ID檢查項目說明嚴重度
ai-patient-requiredpatient.reference 格式必須格式為 Patient/<id>ERROR
ai-code-requiredcode 存在必填，標識過敏原ERROR
ai-category-invalidcategory[*] 值域只允許 food、medication、environment、biologicERROR
ai-criticality-invalidcriticality 值域只允許 high、low、unable-to-assessERROR
Encounter（就醫紀錄）
病人一次就醫事件（門診、住院、急診等）。
Rule ID檢查項目說明嚴重度
enc-status-requiredstatus 必填且值域正確必填（planned/in-progress/finished 等）ERROR
enc-class-requiredclass.code 存在必填，就醫類別代碼（AMB 門診、IMP 住院、EMER 急診）ERROR
enc-subject-requiredsubject.reference 格式必須格式為 Patient/<id>ERROR
enc-period-start-missingperiod.start 存在建議填寫就醫開始時間WARNING
Procedure（醫療處置）
執行過的醫療處置或手術。
Rule ID檢查項目說明嚴重度
proc-status-requiredstatus 必填且值域正確必填（completed/in-progress 等）ERROR
proc-code-requiredcode 存在必填，標識醫療處置類型ERROR
proc-subject-requiredsubject.reference 格式必須格式為 Patient/<id>ERROR
台灣 NHI 特有規則
針對台灣健保資料的額外驗證，適用於 EHR-FHIR-BRIDGE 產生的資源。
Rule ID適用資源檢查項目說明嚴重度
tw-nhi-id-format Patient 身分證格式 10 碼 identifier：本國籍 ^[A-Z][12]\d{8}$，外籍居留 ^[A-Z][89]\d{8}$。醫院病歷號可忽略此警告。 WARNING
tw-nhi-lab-code-format Observation 醫令代碼格式 urn:oid:nhi.lab.code 應符合 ^\d{4,6}[A-Z]$（如 09001C） WARNING
tw-nhi-med-code-format MedicationRequest 健保藥品代碼格式 urn:oid:nhi.medication.code 應符合 ^[A-Z]{1,3}[0-9A-Z]{7,9}$（含舊式 G0 結尾，如 AC346701G0） INFO
tw-timezone-missing 所有資源 時區標示 日期時間欄位應包含 +08:00 時區（如 2024-01-01T08:00:00+08:00），避免跨時區解讀錯誤 WARNING
tw-meta-source 所有資源 資料來源標記 meta.source 應為 ehr-fhir-bridge/scraper；非本橋接器產生的資料可忽略 INFO
跨資源完整性（批次驗證）
當上傳多個資源時，驗證器會額外檢查資源間的參照關係是否一致。
Rule ID檢查項目說明嚴重度
ref-patient-dangling Patient reference 可解析 subject.reference = Patient/X 中的 Patient X 必須存在於這批資料中，否則為懸空參照 ERROR
ref-obs-dangling DR→Obs reference 可解析 DiagnosticReport.result[].reference 指向的 Observation 必須存在於這批資料中 WARNING
支援的資源類型

以下資源類型有完整的結構驗證。其他類型的資源仍會檢查基本欄位（id、台灣 NHI 規則、參照完整性），但不會驗證特定欄位。
